一、项目定位

一个 单进程、零外部依赖 的 AI 消息平台，后端集成 QQ(NapCat) 和 Web 管理后台。用户可以创建多个 Agent（角色/人设），LangChain 编排 LLM 调用，SQLite 存储全部数据。

架构原则
  • 单进程 Express 服务，无需 Docker / K8s
  • 存储仅依赖 sql.js（纯 JS SQLite）+ 文件系统 data/ 目录
  • LLM 框架: LangChain (ChatOpenAI)
  • 零外部中间件: 无 Redis / RabbitMQ / Milvus / MinIO

    三、核心数据流

    QQ 消息 → NapCat WebSocket
  → agentManager.handleMessage(msg)
    → traceManager.trace('agent.handleMessage', ...)
    → getSession(userId).currentAgent       ← 记忆模块
    → matchAgent(msg)                       ← 路由匹配（command > keyword > catchall）
    → buildSystemMessages(agent)            ← 含人设渲染
    → getContextMessages(userId)            ← 短期记忆（最近 20 轮）
    → createModelForAgent(...)              ← 模型工厂
    → resolveTools(agent)                   ← MCP 注册表筛选 Agent 可用工具
    → model.bindTools(tools).invoke()       ← LangChain 函数调用
    → 自动执行 tool_calls 循环
    → addMessageToContext(assistant, text)  ← 写入记忆
    → 回复 → NapCat sendMsg

    十、常见问题

    Q: EADDRINUSE 端口被占用？
    A: 程序会自动尝试下一个端口（3000→3001...），或手动 Get-NetTCPConnection -LocalPort 3000 | Stop-Process

    Q: 知识库向量化失败？
    A: 需要本地运行 Ollama 并加载 bge-large-zh-v1.5 模型，首次 embed 有 3 秒超时。也可留空跳过向量化，走全文搜索回退。

    Q: 如何接入真实搜索？
    A: 修改 knowledge/tools/index.ts 中 web_search 工具的 execute 实现，接入 SerpAPI / Bing Search API。

    Q: tsx src/index.ts 无输出？
    A: 一些环境下 tsx 有 stdout 缓冲问题，使用 npx tsc && node dist/index.js 替代。
